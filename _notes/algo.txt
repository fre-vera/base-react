1. в таск трекере создаем и оформляем задачу
2. создаем ветку для решения задачи
3. локально переключаемся на ветку задачи
4. в папке для хранения решений задач (js) создаем папку для решения конкретной задачи taskName
5. в папке для решения задачи, создаем файл для решения задачи taskName.js
6. в папке для решения задачи, создаем файл для описания задачи README.md
7. в папке для решения задачи, создаем файл для типов (опционально) types.ts
8. в папке для решения задачи, создаем файл для тестов решения задачи taskName.test.js
9. заполняем файл README.md
10. заполняем файл для тестов
11. заполняем файл для типов
12. в файл для решения задачи импортируем типы из файлов типов с помощью JSDoc комментария
13. описываем мини документацию функции (решения задачи) с помощью JSDoc комментария
14. определяем функцию (решение задачи)
15. объявляем параметры функции (опционально)
16. обдумываем условие задачи
17. собираем в конспект всю необходиммую информацию для решения задачи (опционально)
18. в теле функции в виде комментариев построчно описываем аллгоритм решения задачи (пвседо код)
19. каждый элемент алгоритма на пвседо коде заменяем синтаксической инструкцией
20. запускаем тест (прогоняем через тесты) решение задачи
21. если тесты пройдены и отсутвуют замечания линтеров (синтаксис и типов) то задача решена
22. коммиты и отправка в удаленный репозиторий, PR на проверку
23. если апрув, то мердж и закрытие задачи
24. обновляем локальную главную ветку
25. переходим к решению следующей задачи
